## 二、V1.4 接力期关键设计(5 个 m8t)

### 2.1 m8t1:30 脚本聚合回归

设计要点:
- **聚合而非重写**:沿用 m7t1_test_regression.py 的 21 脚本(M5+M6),串联 m7t2~m7t10 的 9 个脚本,合计 30 脚本
- **串行执行**:沙箱不并行(避免子进程抢资源)
- **超时 180s/脚本**:失败也继续跑完
- **退出码语义**:全部通过返 0,任意失败返 1
- **m7t3 跳过旧版**:只跑 `m7t3_test_dataset_real.py`(本轮新建),跳过 `m7t3_test_real_dataset.py`(旧版)

### 2.2 m8t2:合规 + OpenAPI drift 双校验

设计要点:
- **CR-010 全文扫描**:9 禁词 + 3 模式全过(frontend/src + backend/app)
- **docs/ 豁免**:M*-handoff.md 需记录 CR-010 禁词清单供 reviewer 参考
- **OpenAPI v1.5 freeze 校验**:48 endpoints / 40 paths / 13 tags,admin 4 端点 + webhook 端点存在,EDGAR 端点不暴露
- **PYTHONIOENCODING=utf-8**:子进程设环境变量,避免 PowerShell GBK 截断 emoji(✅/❌)

### 2.3 m8t3:28 项环境变量分 P0/P1/P2/P3

设计要点:
- **P0 7 项必填**:Database / SECRET_KEY / Stripe 4 项(支付链路)
- **P1 8 项必填**:VAPID 3 项 / Sentry / ENV / DEBUG / LOG_LEVEL / CORS_ORIGINS
- **P2 7 项建议**:Redis / Google OAuth / FINRA / SEC(沿用默认即可)
- **P3 9 项**:全部沿用 config.py 默认值
- **沙箱 fallback 显式标注**:SECRET_KEY 含 `dev-only-` 前缀,Stripe/VAPID 默认 None
- **12 项落地检查清单**:CI 自动 10 项 + 人工 2 项

### 2.4 m8t4:7 步上线 + 4 级 rollback

设计要点:
- **7 步上线清单**:secrets → database → redis → vapid → 双签 → ETL → 重测
- **灰度切流 4 阶段**:0% / 10% / 50% / 100%,72h 观察窗口
- **Rollback 4 级**:
  - L1 软回滚(5min,配置层面)
  - L2 数据回滚(30min,切回 sandbox stub)
  - L3 代码回滚(1h,git revert V1.3.0)
  - L4 紧急停服(5min,maintenance mode)
- **监控 6 健康指标 + 4 业务指标**:API 错误率 / 延迟 / 推送送达 / Webhook 签名 / Stripe 成功率 / DB 池利用率
