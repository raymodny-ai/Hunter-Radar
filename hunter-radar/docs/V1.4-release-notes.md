## 三、关键功能详解(7 项)

### 3.1 Threat Score 校准前默认权重

**Stock**:
- options: 30%
- short: 35%
- divergence: 20%
- insider: 15%

**ETF**:
- options: 35%
- short: 45%
- divergence: 20%

**校准状态**:v3.0-final 重测 Mann-Whitney U p=0.3827,不显著;保持 v1.0 默认权重。

**红警阈值**:`THREAT_RED_THRESHOLD=70`,`THREAT_RED_THRESHOLD_PANIC=80`(VIX>30 / SPX<MA20 时)

### 3.2 5 段雷达

5 段时间窗:7D / 30D / 90D / 1Y / All

每段独立 Threat Score + 信号分布柱图,前端通过 SSE 实时刷新。

### 3.3 Ultimate Alert Overlay

命中 `THREAT_RED_THRESHOLD_PANIC=80` 时全屏弹窗,音频 + 视觉 + 推送三通道并发。

CR-010 兜底:弹窗必含「仅供统计 / 参考 / 历史数据」三段文案。

### 3.4 Pro 订阅

**价格**:
- 月付:$19 USD(`STRIPE_PRICE_PRO_MONTHLY`)
- 年付:$188 USD(`STRIPE_PRICE_PRO_YEARLY`)

**免费配额**:3 次/天(雷达查询 / 选股器筛选 / 详情页访问合计)

**Pro 配额**:9999 次/天(实际无限)

**支付**:
- Stripe Checkout 跳转支付
- Webhook 签名校验(`signature_mode: prod_verified`)
- 三态:sandbox_skip / prod_unavailable / prod_verified

### 3.5 8-K Item 8.01 实时聚合

**数据源**:EDGAR full-text search(沙箱 stub,生产 httpx + EDGAR API)

**4 类别**:
- share-repurchase(股票回购)
- material-agreement(重大合同)
- press-release(新闻稿)
- other(其他)

**推送**:3 事件级别(关键 / 重要 / 关注) + 4 推送通道(站内 / 邮件 / WebPush / webhook)

### 3.6 PWA + Lighthouse

**Workbox**:
- 前端 Service Worker,缓存策略 stale-while-revalidate
- 离线访问已浏览页面

**Lighthouse 目标**:Performance / Accessibility / Best Practices / SEO 四项均 ≥ 90

**VAPID 推送**:
- 私钥:`VAPID_PRIVATE_KEY`(生产单独生成,严禁用沙箱)
- 公钥:`VAPID_PUBLIC_KEY`(前端 Vite 注入)
- 邮箱:`VAPID_CLAIMS_EMAIL`

### 3.7 OpenAPI v1.5 freeze

**端点数**:48 endpoints / 40 paths / 13 tags

## 六、Rollback 步骤(4 级)

### 6.1 L1 软回滚(配置层面,5 分钟)

**触发条件**:VAPID / Sentry / Stripe webhook 任一 silent degradation

**操作**:
1. 暂停灰度(`vercel --target=staging` 或 Render 切回 staging)
2. 注入 fallback 环境变量
3. 验证 `signature_mode: prod_verified` → `sandbox_skip`
4. 通知 ops + CR + 产品

### 6.2 L2 数据回滚(回滚 ETL,30 分钟)

**触发条件**:数据完整性失败(行数 ≠ 4220 / SHA256 不一致)

**操作**:
1. 停止 `backfill_pg` 进程
2. 切回 `backtest_dataset_real.py`(沙箱 stub)
3. `review_mode` 切回 `sandbox_stub`
4. 通知 CR 走双签流程
5. 排查 ETL 失败原因

### 6.3 L3 代码回滚(切回 V1.3,1 小时)

**触发条件**:严重 bug 导致 Sentry 错误率 > 5%

**操作**:
1. `git revert` 切回 V1.3.0 tag
2. CI 6 jobs 实跑 + pytest 194 passed
3. 重新部署 V1.3.0
4. 通知用户(站内公告 + 邮件)
5. 排查 V1.4 失败原因

### 6.4 L4 紧急停服(立即停,5 分钟)

**触发条件**:数据泄露 / 安全漏洞 / 法律合规问题

**操作**:
1. 立即停服(Render / Vercel 切 maintenance mode)
2. 通知法务 + CR + 产品
3. 启动安全事件响应流程
4. 24h 内提交事故报告
