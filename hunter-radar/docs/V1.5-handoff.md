## 二、V1.5 接力期关键设计(8 个 m9t)

### 2.1 m9t1:Admin 鉴权三重防护

设计要点:
- **JWT role 校验**:`require_admin` 依赖,检查 JWT 中 `role=="admin"`
- **ADMIN_API_KEY 备选**:`X-Admin-API-Key` header,绕过 JWT(紧急运维用)
- **IP 白名单**:`ADMIN_IP_WHITELIST` 环境变量,逗号分隔 IP/CIDR
- **sandbox fallback**:无 JWT / 无 API_KEY / 无 IP 白名单时 → 显式 `review_mode=sandbox_skip_admin` + `sandbox=true`
- **严禁 mock 200 伪装**:任何路径若 JWT 校验失败 → 401(而非 200+空)

### 2.2 m9t2:env_check.py 自动化

设计要点:
- **CI 10 项自动检查**:SECRET_KEY / DATABASE_URL / STRIPE_* / VAPID_* / SENTRY_DSN / CORS_ORIGINS
- **V1.5 新增 3 项**:ADMIN_API_KEY / ADMIN_IP_WHITELIST / SECRET_KEY 长度
- **人工 2 项**:PG 三扩展 / VAPID 公私钥配对
- **sandbox 适配**:开发环境 SECRET_KEY 含 `dev-only-` 前缀不报警(仅警告)
- **退出码语义**:全部 PASS 返 0,任一 FAIL 返 1

### 2.3 m9t3:BD-086 双签 reviewer CLI

设计要点:
- **6 子命令**:register / list / revoke / sign / verify / batch-verify
- **HMAC-SHA256 token**:`nonce + sig`[:128] 字符 hex,SECRET_KEY 派生
- **env var 隔离**:`REVIEWER_CLI_DATA_DIR` 支持自测用 tmp_path,不污染主数据目录
- **31 事件签核**:每个事件需 2 个 reviewer 签核(`prod_signoff_v1`)才进生产
- **batch-verify**:扫 31 事件 goldset,统计已签 / 待签 / 已拒绝
- **GBK 编码兼容**:verify() 输出 `[OK]` 而非 `\u2713` emoji(避免 PowerShell GBK 崩溃)

### 2.4 m9t4:EDGAR fulltext 端点暴露

设计要点:
- **2 端点**:`GET /api/v1/edgar/search` + `GET /api/v1/edgar/categories`
- **6 query 参数**:query / tickers(逗号分隔)/ from_date / to_date / category(Literal 4 类)/ limit(1-50)
- **沙箱 stub**:无 EDGAR_API_KEY 或 httpx 不可用 → `fetch_fulltext_sandbox`(确定性合成)
- **4 类 category 与 eight_k.py 同步**:share-repurchase / material-agreement / press-release / other
- **响应字段**:`summary` + `filings[]` + `sandbox=true` + `review_mode="sandbox_stub"` + `query_meta` + `disclaimer`
- **R-36 解除**:EDGAR 公开端点(v1.5 之前 etl 层不暴露)

### 2.5 m9t5:ETF 申赎代理 3 端点

设计要点:
- **3 端点**:`GET /api/v1/etf/basket` + `POST /api/v1/etf/orders` + `GET /api/v1/etf/premium-discount`
- **Pydantic EtfOrderRequest**:5 字段(etf / order_type / settlement_mode / units / ap),字段长度 + 范围 + 字面值校验
- **field_validator**:order_type ∈ {creation, redemption};settlement_mode ∈ {cash, in_kind}
- **错误码**:`400`(参数错)+ `422`(字段值不合法)
- **BD-088 sandbox_stub_v15_prep**:订单 submit 仅构造对象,不发真实 AP 请求
- **premium-discount**:计算 `premium = market_price - nav` + `arb_opportunity` 套利窗口

### 2.6 m9t6:Analytics events 3 端点

设计要点:
- **3 端点**:`GET /api/v1/analytics/events` + `GET /api/v1/analytics/funnel` + `GET /api/v1/analytics/event-names`
- **10 事件名**:`ALL_EVENT_NAMES` tuple 锁定(V1.5 spec 沿用)
- **时间范围过滤**:from_ts / to_ts 支持 ISO 8601 UTC(含 `Z` 与 `+00:00`)
- **in-memory ring buffer**:沙箱用(maxlen=1000),重启后丢失
- **PII 脱敏**:`hash_user_id` 用 SHA256(沿用 V1.5 spec)
- **订阅漏斗**:`signup → subscribe_start → subscribe_success` 三阶段转化率
- **sandbox_stub_v15_prep**:显式标注,生产前需接 postHog / Plausible / ClickHouse(R-39 已知)

### 2.7 m9t7:OpenAPI v1.5.1 freeze

设计要点:
- **dump 脚本**:`m9t7_dump_openapi_v151.py` 静态扫 `app/api/*.py` 全部 `@router.{get,post,put,delete}`
- **ROUTER_PREFIX 映射**:16 router 各自的 prefix(health 空 / edgar /api/v1/edgar 等)
- **沙箱 fallback 纯 AST**:不 import app.main,无 fastapi / sqlalchemy / sentry_sdk 仍可跑
- **freeze 输出**:`docs/openapi-frozen-v1.5.1.json`(机器可读)+ `docs/openapi-frozen-v1.5.1.md`(人手增补)
- **FE-010 同步**:`FE-010-changelog-v1.5.1.md`(295 行,前端 openapi-typescript 重生成)
- **新增 3 tag**:edgar / etf / analytics(13 → 16)
- **新增 8 endpoints**:EDGAR 2 + ETF 3 + Analytics 3(48 → 56)
- **R-36 解除**:EDGAR 公开;R-34/35 部分解除:admin 鉴权由 m9t1 落
