## 二、V1.5.2 增量特性

### 2.1 真实数据源接入(3 项 P0)

#### m10t1 EDGAR 真实 SEC API
- 入口:`backend/etl/edgar_real.py` → `fetch_fulltext_sec_httpx`
- 协议:`https://efts.sec.gov/LATEST/search-index` (HTTP GET)
- 必填:`SEC_API_USER_AGENT="AppName admin@example.com"`(SEC 强制)
- 频率限制:`SEC_API_RATE_LIMIT_DEFAULT = 0.15s` 间隔
- 超时:`SEC_API_TIMEOUT_DEFAULT = 15.0s`
- 4 fallback 路径:httpx_unavailable / user_agent_invalid / 4xx-5xx / httpx_error
- 响应字段增强:`fetch_source="sec_httpx" | "sandbox_stub"` + `http_status` + `latency_ms` + `warning`

#### m10t2 ETF 真实 AP 代理数据源
- 入口:`backend/app/services/etf_proxy_real.py` → `fetch_etf_proxy_indicators`
- 协议:yfinance 公开代理指标(异步 + asyncio.wait_for)
- 指标:`market_price` / `nav` / `inav` / `premium` / `premium_pct` / `inav_deviation`
  / `volume_5d_avg` / `volume_30d_avg` / `volume_spike_ratio` / `arb_opportunity`
- 套利窗口阈值:`|premium_pct| > 0.5%` OR `|inav_deviation| > 0.3%` → `arb_opportunity=True`
- 放量阈值:`volume_spike_ratio > 2.0` → 异常放量
- 4 fallback 路径:yfinance_unavailable / yfinance_timeout / yfinance_error / provider_sandbox
- 端点变更:`premium-discount` 的 `price` 参数从必填改可选(price 缺省时走 yfinance 真实代理)

#### m10t3 Analytics 真实事件库接入
- 入口:`backend/app/services/analytics_real.py` → `fetch_analytics_real_events`
- 双 provider:postHog(优先,`https://us.i.posthog.com`)/ Plausible(次选,`https://plausible.io`)
- 必填:`POSTHOG_API_KEY` / `PLAUSIBLE_API_KEY` + 可选 `POSTHOG_PROJECT_ID` / `PLAUSIBLE_SITE_ID`
- 4 fallback 路径:httpx_unavailable / provider_sandbox / unknown_provider / posthog_error
- 标准化:10 事件名 + hash_user_id(SHA256) PII 脱敏
- 响应字段增强:`fetch_source` + `http_status` + `latency_ms` + `warning`

## 五、解除 freeze 流程(沿用 V1.5 / V1.5.1)

V1.5.2 freeze 是 v1.5.1 增量 freeze,**不重启** OQ-01 / OQ-02 / OQ-16 / CR-010:

- **新增端点**:必须在 freeze 解除流程中登记,不得私下增路由(本期 0 新增)
- **修改端点**(字段 / 路径 / 状态码 / 错误体):必须先 freeze 解除,改完再 freeze(本期 3 端点响应增强)
- **删除端点**:必须先 freeze 解除,标 deprecated 90 天后才物理删除(本期 0 删除)
- **数据缺失语义**:统一返 200 + 空数组 / 空对象,严禁 5xx 透传
- **沙箱 fallback 显式标注**(M9 加固,V1.5.2 沿用):所有 V1.5.2 端点必须含 `fetch_source` + `sandbox=true` + `review_mode` 字段
