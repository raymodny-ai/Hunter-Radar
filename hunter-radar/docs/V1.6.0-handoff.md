## 5. 五大优化详情

### 5.1 数据管道与健壮性 (P0)

- **多源冗余**: MarketDataProvider 抽象接口 + YFinance/AlphaVantage 双源 + DataProviderManager 三级降级
- **数据校验**: 4 类校验(daily_price/short_volume/form4/options_chain)，标记异常不丢弃
- **统一重试**: tenacity 指数退避(attempts=3, wait=5s~60s)，`etl_retry` 装饰器 + `run_stage_with_retry()`

### 5.2 量化计算模型 (P1)

- **ML 动态权重**: LinearRegression 计算各模块 R² → 归一化权重 → clamp [0.10, 0.50]，冷启动兼容(<90天回退默认)
- **VWMA 去噪**: 成交量加权移动平均平滑做空比例 + 融资余额互证(做空↑ + 融资↓ → 降低系数)

### 5.3 系统架构与性能 (P1)

- **物化视图**: `mv_screener_top100` 预计算 Top 100，`REFRESH CONCURRENTLY` 由 pipeline §9 触发
- **归因分析**: 各模块 weight × score 贡献 + 瀑布图数据 + API 端点
- **环境时间轴**: 逐日 regime 计算 + 切换点标注(normal↔panic)

### 5.4 LLM RAG 知识库 (P2)

- **知识构建**: 从 form4_event/buyback_event/edgar_fulltext 提取 → sentence-transformers embedding → pgvector 存储
- **知识检索**: 向量相似度(top-K=5) + 降级为时间排序
- **Prompt 增强**: 历史文档注入 LLM system prompt context

### 5.5 Docker 容器化 (P2)

- **多阶段构建**: base → builder(pip install) → runtime(non-root + healthcheck)
- **compose 编排**: postgres + redis + backend + etl-cron + airflow，hunter-net 内部网络
- **控制脚本**: start/stop/restart/logs/migrate/seed/status 命令

## 6. ONLINE-READY 结论

V1.6.0 通过 73 脚本 / 1,602 测点全量验证，5 大优化领域全部完成，系统达到 ONLINE-READY 状态。
