## 五、4 项优化微调落地详情

### 微调 1: Options 动态基准 + 防风控限流

- `DynamicBaseline` dataclass: ETF floor=1000 vol×3.0 / Stock floor=200 vol×5.0
- `compute_dynamic_vol_min(ticker_type, avg_30d_volume)` 替代硬编码
- `options_cron.py`: shuffle tickers + jitter 0~120s + rate limiter 1.5/s + DTE 0-7

### 微调 2: Threat Score 动态权重重分配

- `reallocate_weights(signals, base_weights)`: HIGH 模块 → 0.40, 剩余按原比例压缩
- `compute_threat_score` 返回 `min(ema_today, 100.0)` 硬截断
- `load_threat_score._read_options_signals`: Redis → option_pcr_daily → 默认 NORMAL

### 微调 3: Playwright 资源管理 + Fallback 告警

- `async with async_playwright() as p` + `finally: browser.close()` 双重保险
- `check_fallback_streak`: 连续 ≥ 3 天 fallback → `log.warning`
- UA 池 3 个浏览器字符串轮换

### 微调 4: Redis TTL=40min + Cron 缓存预热

- `warm_options_cache`: ETL 完成后 SET `opt:{ticker}:{date}` TTL=2400s
- `warm_ats_cache`: Cron 预热 ATS 数据 TTL=21600s
- API 端点 `/options-anomaly-v2`: 纯 Redis 读, cache miss 返回空对象
- API 端点 `/short-iceberg-v2`: 合并主源 + ATS 系列
