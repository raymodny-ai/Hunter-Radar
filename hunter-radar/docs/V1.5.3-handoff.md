- 不得直接改 `backend/app/api/*` 不经评审

## 五、关键设计原则(沿用 V1.5 / V1.5.2)

1. **数据缺失返 200+空,严禁 mock 200 伪装** — V1.5 production 锁定
2. **沙箱 fallback 显式标注** — 8 值 `fetch_source` + 3 值 `auth_mode`
3. **三态 admin 鉴权** — `prod_admin_jwt` / `prod_admin_apikey` / `sandbox_skip_admin`
4. **静态自测优先** — 沙箱 stuck 时,走纯文本正则/字段校验
5. **OpenAPI freeze 沿用 V1.5.2** — 评审通过项 0 个端点变更
6. **V1.5.3 评审通过 7 项** — 修复均不引入新端点,只在内部实现层
7. **runtime 拦截** — m7t2 等弃用脚本默认拒绝,需 `HR_ALLOW_LEGACY_SCRIPTS=1` 放行
