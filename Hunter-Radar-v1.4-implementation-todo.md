## 1. 项目总览

### 1.1 交付目标
- **一期上线(7.5 周内)**:完成 PRD §1–§4 全部核心能力,前后端打通,支持免费版 + Pro 订阅;含 Threat Score 回测校准与 EMA 防毛刺。
- **二期扩展(独立项目)**:机构版 API、ETF 一级市场申赎数据接入、历史回测产品化。

### 1.2 团队编制建议
| 角色 | 人数 | 备注 |
|---|---|---|
| 后端 ETL / 数据工程 | 1–2 人 | Python / Airflow / SQL 为主,需熟悉监管数据源 |
| 前端工程 | 1–2 人 | React / TS / 可视化(沿用 frontend-plan 技术栈) |
| 合规与风控 | 0.5 人 | 兼任,可由产品法务对接 |
| 产品 / 运营 | 0.5 人 | 兼任,需关注商业化节奏 |

### 1.3 优先级判读原则
- **P0**:不交付即无法上线,或属于 PRD 中明确描述的功能 / 合规硬要求。
- **P1**:增强体验、可显著提升转化或降低运营成本。
- **P2**:可在 V1.5 / V1.6 兑现,或属于商业化二期内容。
